# YLJ5 正演仿真与运行模型

本文描述当前 `YLJ5 / AGHJ-I-LIDAR(MPL)` 仿真设备，不再描述其他厂家或通用批处理链。

## 1. 默认观测计划

```text
周期 0：90 度垂直观测 1 条 -> 0 度水平扫描 180 条
周期 1：90 度垂直观测 1 条 -> 5 度锥形扫描 180 条
随后按 0/5 度轮换
```

| 参数 | 默认值 | 口径 |
|---|---:|---|
| 方位范围 | 0-360 度 | 公开能力边界内 |
| 方位步进 | 2 度 | 满足至少 180 条 |
| 方位射线驻留 | 2 秒 | 为满足积分脉冲数选择的假设 |
| 云台速度 | 10 度/秒 | 低于 30 度/秒边界 |
| 垂直观测驻留 | 30 秒 | 仿真假设 |
| 当前方位层 | 0 或 5 度 | 自动顺序和 5 度均待实机确认 |
| 距离门 | 5334 x 3.75 m | 覆盖 20002.5 m |

方位扫描约 406 秒，包含垂直观测的周期约 436 秒。服务端播放倍率只缩短等待时间，
不会修改正演中的驻留时间和积分脉冲数。

## 2. 大气场景

正演真值由分子背景、边界层气溶胶、高架输送层和风驱动污染羽流组成。相对湿度影响
吸湿增长，风速和风向影响烟羽平流，时间相位影响边界层和背景光。`construction_site`、
`urban_grid` 和 `mobile_mapping` 是仿真场景标签，不是厂家设备模式。

真值只用于生成回波和测试。实时协议默认不发送 `true_backscatter`、`true_extinction`、
`true_pm25`、`true_pm10` 或热点掩膜。

## 3. 光电正演

每个距离门使用离散 LiDAR 方程：

```text
P(r) = C * E * O(r) * beta(r) * exp(-2 * tau(r)) / r^2 + background
```

模型包含太阳背景、暗计数、读噪声、Poisson 计数涨落、大信号高斯近似、死时间软压缩、
后脉冲和饱和上限。输出单位为 `mean_counts_per_pulse`，积分脉冲数由 `PRF * dwell` 得到。

这些噪声参数用于构造可重复数据，不是 YLJ5 实测噪声曲线。

## 4. 双望远镜四通道

每条射线生成近场/远场和两种偏振组合：

| 通道 | 内容 |
|---|---|
| `near_parallel_532nm` | 近场平行偏振计数与 overlap |
| `near_perpendicular_532nm` | 近场垂直偏振计数与 overlap |
| `far_parallel_532nm` | 远场平行偏振计数与 overlap |
| `far_perpendicular_532nm` | 远场垂直偏振计数与 overlap |

外层 `raw_counts` 由近、远场平行通道平滑拼接，作为实时反演主通道。近远场相对增益、
重叠曲线和约 150 m 拼接尺度都等待实机标定。协议中的体退偏比也未经过偏振串扰矩阵
校正。

## 5. 数据和时序

新连接先收到一次 `status`。每周期开头发送 `telemetry`，随后按采集顺序发送全部
`lidar_raw`，再发送可选 `camera`、`lidar_product`、`ground_obs`，最后由服务器发送
`heartbeat`。

- `camera` 只声明同步能力、当前指向和无图像原因；
- `lidar_product` 只提供未标定信号与退偏比摘要；
- `ground_obs` 默认关闭，开启后仍明确是合成辅助观测；
- `telemetry` 标记为 `synthetic_not_vendor_spec`。

当前 `jsonl-ylj5-emulator-1.0` 是项目测试协议，不是厂家线协议。

## 6. 惰性生成

初始化阶段只校验公开规格。扫描时一次生成一个周期，再逐帧序列化发送，避免缓存
180 个周期的四通道数组。`produce_scan_cycle` 仅供测试收集整周期；正式服务使用
`stream_scan_cycle`。

若正演失败，设备进入 `fault` 并发送 `alarm`；若客户端断开、设备停止或进程退出，
当前周期中止且不发送完成心跳。

## 7. 可验证边界

当前可由测试直接验证：

- 默认规格配置无违规项；
- 0/5 度方位层按周期轮换；
- 单圈 180 条、每条 10000 个积分脉冲；
- 5334 距离门和四通道序列化一致；
- 低于公开采购边界的在线扫描配置被原子拒绝；
- 暂停、恢复、停止和状态查询有确定响应。

无法仅靠当前代码验证：厂家帧格式、实机噪声统计、标定曲线、默认扫描顺序、相机数据、
状态寄存器和厂家 L1/L2 产品。
