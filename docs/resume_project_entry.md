# 简历项目版本

## 项目名称

YLJ5 颗粒物激光雷达公开规格数字孪生与 Linux Qt 实时客户端

## 一句话描述

基于公开产品资料、采购技术边界和扫描雷达论文，实现 YLJ5 四通道 C++ 仿真设备、
实时 JSONL 链路及 Linux Qt 子线程处理客户端，并对未取得实机证据的协议和标定参数
进行显式隔离。

## 简历条目

- 梳理 `YLJ5 / AGHJ-I-LIDAR(MPL)` 公开证据，建立“官网确认、采购边界、仿真假设、
  实机未知”保真矩阵，并把 532 nm、3.75 m、20 km、双望远镜、偏振和云台约束固化为
  可执行配置校验。
- 使用 C++20 构建双望远镜平行/垂直偏振四通道正演，模拟 LiDAR 方程、Poisson 噪声、
  背景、暗计数、死时间、后脉冲和饱和；按周期惰性生成 5334 距离门数据，避免整场缓存。
- 实现 90 度垂直观测与 0/5 度方位层轮换、采集驻留时序、设备状态机、POSIX TCP 和
  JSONL 控制协议；非法在线扫描配置以完整违规列表原子拒绝。
- 将 Qt 客户端网络、拆帧、周期完整性、近远场拼接、Fernald/Klett 反演、湿度修正、
  ENU 投影和显示快照全部放入专用 `QThread`，GUI 主线程只负责绘制和交互。
- 建立 PM 标定门控：无目标站点共址标定时仅输出光学产品和 QC，不生成伪 PM 或浓度
  热点；通过四项 CTest 覆盖规格、时序、协议、重连状态释放和处理边界。

## 技术栈

C++20、CMake、Qt 6 Widgets/Network、POSIX Socket、TCP JSONL、LiDAR 方程、
Fernald/Klett 弹性反演、偏振退偏、近远场 gluing、ENU 坐标、CTest。

## 面试展开

### 为什么不是 100% 实机克隆

公开资料没有厂家帧、SDK、命令字、标定曲线和产品算法。项目没有用其他厂家格式填空，
而是保留独立仿真协议和 provenance/QC 边界。下一步需要实机抓包与 golden sample 才能
逐项验收。

### 为什么 PM 必须门控

单波长弹性回波先得到依赖激光雷达比和参考边界的光学量，消光到 PM 的关系还受粒径、
成分和湿度影响。没有目标站点共址数据时固定系数会产生虚假定量，因此客户端保持 PM
数组为空。

### 如何控制内存和 GUI 卡顿

服务端一次只生成一个周期并逐帧释放；客户端在工作线程处理网络和数值计算，通过
`shared_ptr<const ...>` 向 GUI 交付不可变快照。周期监控状态在封口后释放，重连时清空。

## 使用边界

简历中可以写“公开规格数字孪生”“仿真协议”“为实机适配预留边界”，不应写“已逆向
厂家私有协议”“已通过实机验收”或引用仿真输出作为设备测量精度。
